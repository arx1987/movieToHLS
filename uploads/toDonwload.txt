#Аббревиатуры Определения
Как расшифровывается аббревиатура REST в контексте API?~representational state transfer
_ - это прикладной программный интерфейс(API), который использует HTTP-запросы для получения, извлечения, размещения и удаления данных~REST API
Если хочешь что-то изменить ты должен бросать UPDATE, PUT, если создаешь - POST, если изменяешь есть варианции PATCH и т.д, если удаляешь - DELETE. Какие принципы описывает предложение выше?~REST~принципы REST~REST принципы
Является ли предложение ниже одним из принципов REST?(да/нет). Единый интерфейс -> ресурсы должны быть однозначно идентифицированы посредством одного URL-адреса и только с помощью базовых методов сетевого протокола(HTTP методы: DELETE, PUT, GET, POST)~да~lf
Является ли предложение ниже одним из принципов REST?(да/нет). Разграничение клиента и сервера -> должно быть четкое разграничение между клиентом и сервером: -пользовательский интерфейс и вопросы сбора запросов - на стороне клиента; доступ к данным, управление рабочей нагрузкой и безопасность - на стороне сервера~да~lf
Является ли предложение ниже одним из принципов REST?(да/нет). Нет сохранения состояния -> любое необходимое управление состоянием должно осуществляться на клиенте, а не на сервере~да~lf
Является ли предложение ниже одним из принципов REST?(да/нет). Кеширование всегда разрешено -> все ресурсы должны разрешать кэширование, если явно не указано, что оно невозможно~да~lf
Является ли предложение ниже одним из принципов REST?(да/нет). Многоуровневая система -> REST API допускает архитектуру, которая состоит из нескольких уровней серверов~да~lf
Является ли предложение ниже одним из принципов REST?(да/нет). Код предоставляется по запросу -> В большинстве случаев сервер отправляет обратно статические представления ресурсов в формате XML или JSON. Однако при необходимости серверы могут отправлять исполняемый код непосредственно клиенту~да~lf
Что предоставляет интерфейс доступа к определенному порту определенного хоста, также есть одноименный класс, который предоставляет низкоуровневый интерфейс для приема и отправки сообщений по сети?~socket~сокет
Верно ли следующее утверждение? (да/нет). Для создания объекта сокета можно использовать один из его конструкторов: 1) Socket(AddressFamily, SocketType, ProtocolType); 2) Socket(SafeSocketHandle); 3) Socket(SocketInformation); 4) Socket(SocketType, ProtocolType);~да~lf
